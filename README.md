# leetcode_arai60

## やり方

- 問題名でブランチを切る
- 問題ごとにフォルダを作成する
- `step1.go`, `step2.go`, `step3.go`, `memo.md`
    - `memo.md` に各ステップで感じたことを追記する

### PRメッセージ

- 問題へのリンク
- 次に解く問題の予告など

## 進め方

[ソフトウェアエンジニアリング協会 LeetCode 部の手引き](https://docs.google.com/document/d/1VUlKFLQJlmWgV8UE8BJ3o6Qy4qXb4E3zeGSz03kR5yE/edit?usp=sharing) より

### Step 1

- 答えを見ずに考えて、5分考えて分からなかったら答えを見る
- 答えを見て理解したと思ったら、答えを隠して書く
- 筆が進まず5分迷ったら答えを見る
- 見たら1回全部消してやり直し

### Step 2

- コードを読みやすくするようにできるだけ整える

### Step 3

- 全部消す
- 時間を測りながらもう一度書く
- 書いてアクセプトされたらもう一度消して書く
- 10分以内に一回もエラーを出さずに書ける状態になるまで続ける

### レビュー

- Step 3 まで終わったら講師陣にレビューを依頼する。
- レビューを元にコードを書き直す
